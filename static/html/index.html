What is AI?

Artificial Intelligence and Machine learning are ways we can use a computer to make a decision. Machine learning is a broad category in computing that contains many different strategies and approaches to decision-making. Specifically, machine learning algorithms allow computers to read large amounts of data and make decisions based on patterns in that data.

Why would we use machine learning to make decisions? Humans are relatively good at making decisions. We make thousands and thousands of decisions a day. However, it is difficult for us to account for all information in unbiased ways. Human brains have a finite amount of active memory that can be used at a time to make a decision which means that we miss things. Sometimes big things.

Let's dive in

... and start with the weather

Machine learning algorithms aim to find patterns in large amounts of data. Say we wanted to predict whether it will rain in the next hour. The machine learning approach to guessing what will happen is to learn from past weather data. Let's assume that for the past year, we took a measurement every hour of the temperature, humidity, altitude and wind speed. Additionally, in each of these hours, we recorded whether or not it rained. A machine learning algorithm figures out how the measurements (temperature, humidity, altitude, and wind speed) are correlated with whether it rained in a given hour. For example, the algorithm may discover that when it is cold and the humidity is high, it is likely to rain, whereas hot and dry conditions did not yield rain as often.

This example may seem as though a human could probably have figured it out without the help of a computer; in this case, that's probably true. The value of machine learning, however, comes from its ability to analyze more data faster than a human and draw conclusions based on more factors than a human could. For weather prediction, a computer might be able to look at more than just humidity and temperature and add global conditions, wind speeds and directions, pressure readings and many more conditions into its decision than a human could.

Let's visualize the computer as the machine to the left; when we put some data into the top of the machine, it drops a prediction out of the bottom. What would go into the top would be the measurements of the current conditions: temperature, humidity, etc.. Within the machine, the data is read and the machine compares it against data it has already seen before. Finally, out of the bottom comes a prediction: whether or not it will rain.

The Decision Boundary

The method by which the current conditions are compared to past data will depend on the algorithm being used. In many machine learning algorithms, the method of coming to a conclusion can be visualized through a decision boundary. A simple example decision boundary can be seen below:

Here, we see a plot of humidity vs. temperature. In this figure, historical data is illustrated by dots: red dots represent hours when rain occurred and green dots represent hours where there was no rain. A machine learning algorithm then learns a 'boundary' between instances where it rained and instances where it did not rain. This boundary is illustrated by the yellow line seen above.

Once this boundary is calculated, it can be used to predict whether it will rain during any given hour. Let's say I make a measurement right now: it is 80 degrees Farienheit and there is a 10% humidity.

We see that these measurements will lie in the 'rainy' section: indicating that it is likely to rain. If we instead, however, measure a 80% humidity, we see that our instance transfers into the 'no rain' section of our graph. This makes sense intuitively, higher humidity indicates a higher likelihood of rain.

Above, we see a fairly simple decision boundary: a straight line. However, we are often able to obtain better predictions using more complex decision boundaries. For instance here, we see that our boundary is able to better differentiate hours where it rained vs. those where it did not.

Additionally, we may have more than one decision boundary; the important aspect is that we are able to separate our graph into, in this case, 'rain' sections and 'no rain' sections.

A little more

Above we asked a simple question - whether or not it will rain. In real world applications, however, we are often faced with more complex problems. Here, we see a few more types of questions that we are able to answer using machine learning. The overall structure of the machine seen above still stays the same, with data going in, and answers given based on the patterns identified within past data.

A binary classification problem categorizes data into a set of predetermined outcomes. For example, the question 'will it rain or will it not rain' is a classification problem because our set of predetermined outcomes are 'yes' or 'no'. Humans naturally solve classification problems like this all the time; for example, you might walk outside and think: 'There are dark clouds in the sky and it feels humid. Usually, when there are dark clouds in the sky and it feels humid, it rains. Therefore, it will likely rain.' Computers using classification algorithms make this exact same determination based on data coming to them from different sources using pressure, humidity and temperature sensors, often far more than a human could keep track of.

In a decision tree, decision boundaries can be set up manually. This means that although they can be easier to grasp and visualize, they are often a bit less accurate in their predictions. Their relative ease of implementation and explanation can make them appealing for implementation.

A neural network is allowed to set up its own connections during training. Although it can sometimes be more difficult to determine the cause of decisions, the decisions can fall into a more tailored series of decision-boundaries that are set by the computer.

A regression problem exists in situations where the prediction falls within a range instead of a specific category. In the weather example, a regression problem would be 'how much is it going to rain?' Here, we cannot create specific categories because it could rain any number of inches. So, given a certain temperature and humidity, the algorithm may predict that 'It will rain 6.34 inches', even if it has never rained exactly 6.34 inches before. The prediction can be more varied because specific categories don't exist.
